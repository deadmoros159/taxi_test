# Настройка DNS для xhap.ru

## 📋 Что нужно настроить

Для работы вашего API нужно добавить DNS записи, которые будут указывать на ваш сервер `102.214.69.160`.

## 🔍 Где настраивать DNS

DNS настраивается в панели управления вашего домена. Обычно это:
- Панель регистратора домена (где вы покупали домен)
- Или панель хостинга, если DNS делегированы туда

### Популярные регистраторы:
- **Reg.ru** - https://www.reg.ru
- **Timeweb** - https://timeweb.com
- **Beget** - https://beget.com
- **Namecheap** - https://www.namecheap.com
- **GoDaddy** - https://www.godaddy.com

## 📝 Какие записи нужно добавить

### 1. Основная A запись для xhap.ru

```
Тип: A
Имя/Хост: @ (или xhap.ru, или оставить пустым)
Значение/IP: 102.214.69.160
TTL: 3600 (или автоматически)
```

**Что это делает:** Указывает, что домен `xhap.ru` должен вести на ваш сервер.

### 2. A запись для www.xhap.ru (опционально)

```
Тип: A
Имя/Хост: www
Значение/IP: 102.214.69.160
TTL: 3600
```

**Что это делает:** Позволяет использовать `www.xhap.ru` (будет редиректить на `xhap.ru` через Nginx).

### 3. A запись для dev.xhap.ru (для development окружения)

```
Тип: A
Имя/Хост: dev
Значение/IP: 102.214.69.160
TTL: 3600
```

**Что это делает:** Создает поддомен `dev.xhap.ru` для тестирования.

## 🖼️ Пример настройки в панели

### Reg.ru
1. Войдите в панель управления
2. Перейдите в раздел "Домены" → выберите `xhap.ru`
3. Откройте "DNS-серверы и зона"
4. Добавьте записи:
   ```
   @    A    102.214.69.160    3600
   www  A    102.214.69.160    3600
   dev  A    102.214.69.160    3600
   ```

### Timeweb
1. Войдите в панель управления
2. Домены → `xhap.ru` → "Управление DNS"
3. Добавьте записи:
   ```
   @    A    102.214.69.160
   www  A    102.214.69.160
   dev  A    102.214.69.160
   ```

### Beget
1. Войдите в панель управления
2. Домены → `xhap.ru` → "DNS записи"
3. Добавьте записи:
   ```
   @    A    102.214.69.160
   www  A    102.214.69.160
   dev  A    102.214.69.160
   ```

## ⏱️ Время распространения DNS

После добавления записей DNS нужно время для распространения:
- **Минимум:** 5-15 минут
- **Обычно:** 30-60 минут
- **Максимум:** до 24-48 часов (редко)

## ✅ Проверка DNS

### 1. Проверка через командную строку

```bash
# Проверка основной записи
dig xhap.ru +short
# Должно вернуть: 102.214.69.160

# Проверка www
dig www.xhap.ru +short
# Должно вернуть: 102.214.69.160

# Проверка dev
dig dev.xhap.ru +short
# Должно вернуть: 102.214.69.160

# Альтернативный способ
nslookup xhap.ru
nslookup dev.xhap.ru
```

### 2. Проверка через онлайн сервисы

- **https://dnschecker.org/** - проверка DNS по всему миру
- **https://www.whatsmydns.net/** - визуальная карта распространения DNS
- **https://mxtoolbox.com/DNSLookup.aspx** - детальная информация

### 3. Проверка на сервере

```bash
# На вашем сервере
curl -I https://xhap.ru
curl -I https://dev.xhap.ru
```

## 🔧 Если DNS не работает

### Проблема: DNS записи не обновляются

**Решение:**
1. Проверьте, что записи сохранены в панели
2. Убедитесь, что TTL не слишком большой (3600 секунд = 1 час)
3. Подождите еще немного времени
4. Очистите DNS кэш на вашем компьютере:
   ```bash
   # macOS
   sudo dscacheutil -flushcache; sudo killall -HUP mDNSResponder
   
   # Linux
   sudo systemd-resolve --flush-caches
   
   # Windows
   ipconfig /flushdns
   ```

### Проблема: DNS указывает на другой IP

**Решение:**
1. Проверьте записи в панели управления
2. Убедитесь, что нет конфликтующих записей
3. Удалите старые записи и создайте новые

### Проблема: Домен не резолвится

**Решение:**
1. Проверьте, что DNS серверы домена настроены правильно
2. Убедитесь, что домен не заблокирован
3. Проверьте через несколько DNS серверов:
   ```bash
   dig @8.8.8.8 xhap.ru
   dig @1.1.1.1 xhap.ru
   ```

## 📱 Быстрая проверка после настройки

```bash
# Через 15-30 минут после настройки DNS выполните:

# 1. Проверка DNS
dig xhap.ru +short

# 2. Проверка доступности через HTTPS
curl -I https://xhap.ru

# 3. Проверка SSL сертификата
openssl s_client -connect xhap.ru:443 -servername xhap.ru < /dev/null
```

## 🎯 Минимальная настройка (только production)

Если вам не нужен dev поддомен, достаточно добавить только:

```
@    A    102.214.69.160
```

Это позволит использовать:
- `https://xhap.ru/auth`
- `https://xhap.ru/driver`
- `https://xhap.ru/order`
- `https://xhap.ru/admin`

## 📞 Нужна помощь?

Если у вас возникли проблемы:

2. Убедитесь, что домен активен и не заблокирован
3. Проверьте логи в панели управления DNS
4. Обратитесь в поддержку вашего регистратора домена
